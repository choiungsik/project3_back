UI(User Interface) 사용자 인터페이스
사용자와 시스템을 원활하게 이어주는 장치나 소프트웨어

CLI(Command Line Interface) - 리눅스기반 텍스트형태로 작동
GUI(Graphical User Interface) - 마우스로 선택하는 그래픽환경
NUI(Natural User Interface) - 사용자의 말이나 행동

UI 기본원칙
직 : 직관성 - 쉽게 이해하고 사용
효 : 유효성 - 목적이 정확하고 완벽하게 사용
유 : 유연성 - 요구사항을 최대한 수용, 실수최소화
학 : 학습성 - 누구나 쉽게 배우고 익힘

가시성 : 주요기능을 메인에 노출시켜 조작이 쉽도록 설계
표준화 : 기능구조와 디자인을 표준화하여 최초 실행 이후에는 쉽게 사용하도록 설계

UI설계도구
와이어프레임(Wirdframe) : 뼈대제작
목업(Mockup) : 실제화면과 유사한 껍데기
스토리보드(Story Board) : 페이지간 흐름 추가(디스크립션)
프로토타입(Prototype) : 실제 구현된 것처럼 테스트 가능한 동적 모형
유스케이스(Usecase) : 사용자 측면의 요구사항

UI작성순서(동환이네 기구)
동환 :구동환경정의
이 :레이아웃정의
네 :네비게이션정의
기 :기능성의
구 :구성요소정의

요구사항 요소(요구는 대기서약 후 이뤄진다)
대 : 데이터요구
기 : 기능요구
서 : 제품/서비스품질
약 : 제약사항

UI프로토타입 장단점
사용자를 설득 및 이해시키기 쉬움
오류 방지 체크
반복된 개선으로 작업시간 증가
필요이상 자원소모가능성 있음

프로토타이핑 종류
- 페이퍼 프로토타입(손으로 직접 작성)
- 디지털 프로토타입(프로그램으로 작성)

UI흐름설계(기입이 유기되면(버려지면) 안되니 흐름을 확인하자)
기 : 기능작성
입 : 입력요소확인
유 : 유스케이스설계
기 : 기능 및 양식 확인
